**SUGGESTION**:
1. **Mapped type mismatch on UUID FK columns** — Several ORM models use `Mapped[str]` for UUID foreign key columns (e.g., `barber_id: Mapped[str]` in `BarberServiceModel`, `AvailabilityModel`, `AppointmentModel`). Since the column type is `UUID(as_uuid=True)`, the correct annotation should be `Mapped[uuid.UUID]`. This works at runtime but misleads type checkers like mypy/pyright. Fix: `from uuid import UUID` and use `Mapped[UUID]` or `Mapped[uuid.UUID]` instead of `Mapped[str]`.
